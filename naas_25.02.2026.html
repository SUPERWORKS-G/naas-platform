0.  Philosophy & Design Decisions

This document defines the MVP architecture for a Notary-as-a-Service platform. It is built on Squarespace, with Airtable as the back-office engine and Zapier connecting everything.

Guiding principles

  • One case, one flow, two views. The client view and the notary view are two faces of the same case record.
  • Clients never see a third-party tool. Every touchpoint is native Squarespace.
  • Airtable stays behind the curtain. It is an operations tool. Not a feature. Not visible to clients.
  • No custom code on critical paths. No JavaScript-driven dynamic pages. No runtime failures.
  • Simple over clever. When two approaches work, choose the one that requires less maintenance.

3.  Case Creation Flow

The flow has two phases. Phase one happens on /start. Phase two happens on /thank-you. The case is not created until the end of phase two, when Stripe payment is confirmed.

3A. Phase One — Intake Form (/start)

The form collects structured data about the client and their request.

How the redirect works

In Squarespace, open the form block settings. Under "Post-Submit", select "Redirect to URL" and enter /thank-you. That is all. When the client submits the form, Squarespace sends them directly to /thank-you. No code needed. This feature is included in the Squarespace Business plan.

Form fields

  • Document Type (dropdown): Power of Attorney / Declaration / Affidavit / Sworn Statement / Consent Declaration for Minors / Mandate
  • Full Name
  • Email Address — used as the client's identity key throughout the platform
  • Phone Number (mandatory) — used for WhatsApp notifications
  • Date of Birth
  • Country of Residence (dropdown: EU countries)
  • Specific Request Details (text area, 500 characters max)
  • Preferred Notary Office (dropdown: onboarded offices, or 'No preference')
  • Three required legal checkboxes (all mandatory to submit):
    • I understand this service does not constitute legal advice
    • I consent to the processing of my personal data as described in the Privacy Policy
    • I understand that a physical notary appointment is required to finalise my documents

3B. Phase Two — The /thank-you Page

This page does the real work. It presents three sequential steps clearly. The client cannot skip any of them.

1
Upload your documents Client clicks "Upload Documents" and is taken to a Google Drive File Request page. They upload their identity document (required) and any supporting files. No Google account needed. No file size limit.
UI note: The upload button is styled natively in Squarespace. The upload itself opens on a Google-branded page — this cannot be avoided with Google Drive. For a fully white-label upload experience embedded directly in the page, File Request Pro (~$49/month) is the recommended upgrade. It renders a branded upload widget inside Squarespace with no Google branding visible.
2
Pay the advance deposit Client clicks the Stripe payment button. They are taken to a Stripe-hosted checkout page to pay the fixed advance deposit (e.g. €50). Stripe handles all payment security. The client is returned to the platform after payment.
3
Select a preferred appointment window A Squarespace form block asks for: first preferred date, second preferred date (optional), preferred time of day (Morning / Afternoon / Evening), and any scheduling notes. This form is submitted independently from payment.

What the client sees on the /thank-you page

"Your request has been received. Please complete the three steps below to open your case. Once your payment is confirmed, you will receive a WhatsApp message with your case reference and a personal link to track your case."

3C. What Happens After Payment — Fully Automatic

How the automation works

When the client completes payment on Stripe, Stripe instantly fires a webhook — a silent, automatic signal — to Zapier. This happens in seconds. No human is involved at any point.

Zapier receives the webhook and executes the following actions in sequence: creates the Airtable case record with all form data, generates the 8-character Client Access Code, creates the Google Drive case folder structure, sends the client a WhatsApp message with their Case ID and magic link, and sends the client a fallback confirmation email.

The client receives their case link within seconds of payment clearing.

Important — Zapier plan requirement

The Stripe webhook trigger is "instant" on Zapier Starter plan (~€28/month) and above. On the free Zapier plan, it would poll every 15 minutes — unacceptably slow. The Starter plan is required for this automation to work as described.

File naming instruction for clients

Ask clients to name their uploaded files using the format: FirstnameLastname_DocumentType (e.g. IonPopescu_Passport.pdf). This makes it easier to sort incoming uploads into the correct case folder. Include this instruction clearly on the /thank-you page.

Design principle

Airtable is a professional operations tool — like email or a spreadsheet. Notary offices and the platform admin use it directly. It is never embedded in Squarespace. Never shown to clients. Never presented as a feature of the platform.

The Status field is the most important field in the system. Every client-facing notification is triggered automatically by a change to this field.

Squarespace email automations only trigger on: form submission, product purchase, or mailing list signup. There is no way to watch a data field and fire a conditional message when it changes. Airtable + Zapier is the only reliable non-code solution for status-driven notifications.

What notaries cannot see

  • Subscription and billing data
  • Admin-only fields

The platform operator has full access to all records. Key admin views:

  • By Notary Office — monitor workload per office

5.  Client Access — Personal Case Pages

Every client gets a private Squarespace page for their case. The page is accessible via a magic link — no login, no account, no password to remember.

The magic link system

Each client's case page URL embeds their unique 8-character access code as a parameter:

acte.online/case-001234?password=ABC12345

When the link is clicked, Squarespace bypasses the password screen automatically. The client lands directly on their case page. Zero friction.

Page architecture

One Squarespace page per case. Pages sit inside a private folder — invisible in site navigation. Each page is password-protected with the client's unique access code. Pages are named /case-XXXXXX to match the Case ID.

Why individual pages instead of a dynamic page

  • Security. Client data is static content in Squarespace, not fetched from an API exposed in the browser. No API key is ever visible to the client.
  • Reliability. A static Squarespace page cannot fail at runtime. A JavaScript API call can.
  • Maintainability. Any non-technical person can update a Squarespace page. No code debugging required.

6.  Notary Access — The Gateway Hub

The Squarespace Member Area is the branded front door for notary offices. The actual case work — reviewing, updating status, accessing documents — happens in Airtable, accessed with one click from inside the Member Area.

The gateway principle

Squarespace provides the interface. Airtable provides the working tool. Each is accessed directly, without embedding one inside the other. Notaries are never asked to work inside an embedded iframe with Airtable's branding showing through.

Why not embed Airtable inside Squarespace?

Iframes in Squarespace cannot be targeted by the site's CSS. Airtable renders with its own fonts, colours, and UI chrome. The result is a jarring visual inconsistency inside what should be a professional platform. The gateway model avoids this entirely.

Notary onboarding — manual at MVP

  1. Platform operator contacts or receives inbound from a notary office.
  2. Admin creates the office's account in Squarespace Members using their primary contact email.
  3. Admin creates a filtered Airtable view for that office. Copies the share link.
  4. Admin sends a welcome email: Squarespace login link, Airtable view link, how-to guide link.
  5. Brief onboarding call recommended — 20 minutes is sufficient.

Why manual onboarding is right for MVP

Up to 10 offices can be onboarded manually with low overhead. Self-service registration adds account approval logic, payment gating, and automated credential delivery — not justified at this stage. Clear upgrade target for Architecture 2.

7.  Notifications — WhatsApp + Email

WhatsApp is the primary notification channel. Email is the fallback. Both are sent automatically by Zapier.

Why WhatsApp over email

Email can land in spam, promotions folders, or simply go unread. WhatsApp delivers directly to the client's screen. For Romanian clients — in Romania and abroad — WhatsApp is the dominant messaging app across all age groups. Delivery is instant. Read rates are far higher than email.

WhatsApp setup

Tool: WhatsApp Business API via Zapier (multiple providers supported, including Twilio)

Cost: Two components. Meta charges ~€0.03–0.06 per conversation (a 24-hour window, regardless of how many messages are sent within it). Twilio adds ~$0.005 per message on top. At MVP volume — 30 cases/month, 3–4 status messages per case — expect €5–15/month in total.

Setup time: WhatsApp Business API approval takes a few days. Apply early — before launch.

Fallback: While WhatsApp Business API approval is pending, all notifications are sent by email only.

Email setup

Tool: Gmail via Zapier

Cost: Included in Google Workspace subscription

Branding: Emails are sent from your platform address (e.g. hello@acte.online). Full HTML template supported — logo, platform colours, Helvetica, single-column layout. Built once in a tool like Stripo (free), pasted into Zapier. Every automated email uses the same branded template.

Deliverability: Configure SPF and DKIM records on your domain before launch to minimise spam filtering.

8.  Payment & Booking

Payment and booking preference are both captured on the /thank-you page — before the case is opened. This is intentional. It commits the client before any notary does any work.

8A. Advance Deposit — Stripe

Tool: Stripe Payment Links (standalone — no Squarespace Commerce required)

Transaction fee: 1.4% + €0.25 per EU card transaction

Recommended deposit: €30–75, fixed, set by the platform operator

How it works

  • Create one Stripe Payment Link for the advance deposit product (e.g. "Notary Appointment Deposit — €50").
  • Embed this link as a styled button on the /thank-you page.
  • The client clicks, is taken to Stripe-hosted checkout, pays, and is returned to the platform.
  • Configure the Stripe Payment Link to collect the client's email address and pass their Case ID as a custom metadata field. This allows Zapier to match the payment to the correct Airtable record.

Stripe vs. Squarespace Commerce

Squarespace Commerce requires a product catalogue, a checkout flow, and a higher-tier plan. A Stripe Payment Link is created in under two minutes. No additional Squarespace plan needed. Full control over refunds and reporting in the Stripe Dashboard.

8B. Appointment Preference Form

Tool: Squarespace form block (native, included in Business plan)

Location: On the /thank-you page, below the Stripe payment button

Form fields

  • Preferred date — first choice (date picker)
  • Preferred date — second choice (date picker, optional)
  • Preferred time of day (dropdown: Morning 09:00–12:00 / Afternoon 12:00–16:00 / Evening 16:00–18:00)
  • Additional notes (optional)

This form is submitted independently from payment. Zapier writes the preferences to the Airtable Preferred Date Range field. The notary reviews these preferences when confirming the appointment slot.

Why not a live calendar?

Romanian notary offices have variable availability — court schedules, walk-in clients, colleagues absent. A live calendar assumes predictable slots. A preference form is more realistic at MVP and avoids double-bookings. Acuity Scheduling can be added as an upgrade when offices are ready for it.

9.  Automation Workflows — Zapier

Recommended plan: Zapier Starter (~€28/month) — 750 tasks/month, sufficient for MVP

Monitoring: Check task usage monthly. Upgrade to Professional if approaching the limit.

Workflow 1 — Case Creation (Triggered by Stripe)

Trigger: Stripe Checkout Session Completed (instant webhook)

  • Find the corresponding Squarespace form submission in Airtable using client email + Case ID from Stripe metadata.
  • Create a new record in the Airtable Cases table. Map all form fields.
  • Generate an 8-character alphanumeric Client Access Code. Store in the record.
  • Set Status = "Submitted", Payment Status = "Paid", Stripe Payment ID = session ID.
  • Create Google Drive folder: /Cases/CASE-ID/Client-Uploads/ and /Cases/CASE-ID/Prepared-Documents/
  • Send client a WhatsApp message with Case ID and magic link.
  • Send client a fallback confirmation email with the same content.
  • If a notary office was pre-selected: send that office a WhatsApp/email notification.

Form data bridging note

The Squarespace form is submitted before payment. Zapier needs to connect the Stripe payment to the correct form submission. The recommended approach: when the client submits the form, Zapier stores the form data temporarily in Airtable as a "pending" record (no Case ID yet). When the Stripe payment fires, Zapier matches by client email and upgrades the pending record to a full case record. Full setup is covered in Annex E.

Workflow 2 — Status Change → Client Notification

Trigger: Airtable record updated (Status field change detected)

  • Filter: only continue if Status has changed.
  • Path A — Status = "In Review": WhatsApp to client: "Your case is being reviewed and prepared by [Office Name]."
  • Path B — Status = "Booked": WhatsApp to client with confirmed appointment date, time, and notary office address.
  • Path C — Status = "Completed": WhatsApp to client with Google Drive document download links.
  • Each path also sends a fallback email with the same content.

Workflow 3 — Booking Confirmation

Trigger: Airtable record updated (Booking Date field updated)

  • Filter: only continue if Booking Date is not empty.
  • Send client a WhatsApp with the confirmed appointment date, time, and notary office address.
  • Send fallback email with the same content.

Workflow 4 — Appointment Preference Captured

Trigger: Squarespace form submission (appointment preference form on /thank-you)

  • Find the matching Airtable case record by client email.
  • Write the preferred dates and time preference into the Preferred Date Range field.

Workflow 5 — GDPR Data Retention

Trigger: Zapier scheduled task (runs daily at 02:00)

  • Find all Airtable records where Status = "Completed" AND Submission Date is older than 12 months.
  • Delete the Airtable case record.
  • Delete the corresponding Google Drive case folder and all contents.
  • Log the deletion in a "Deletion Audit" table: Case ID, deletion date, reason. No personal data stored.

Workflow 6 — Automated Case Page Creation (Optional, MVP+)

Not required at launch. Add at 30+ cases/month when manual page creation becomes a bottleneck.

  • Trigger: New Airtable case record created.
  • Duplicate a template Squarespace case page via the Squarespace API.
  • Set the URL slug to the Case ID and the password to the Client Access Code.

11.  GDPR & Data Compliance

This platform handles sensitive personal data — identity documents, dates of birth, legal declarations. GDPR compliance is a legal obligation. It must be completed before the first live case.

Legal obligation

Romania is an EU member state. The platform operator is a Data Controller under GDPR. Each notary office is a Data Processor. Identity documents require: explicit recorded consent, a clear legal basis for processing, appropriate access controls, and a signed Data Processing Agreement with every notary office.

Required actions before launch

  1. Sign a DPA with every tool: Squarespace, Airtable, Google Workspace, Zapier, Stripe, Twilio/WhatsApp.
  2. Sign a Data Processing Agreement with each notary office.
  3. Document the complete data flow (Record of Processing Activities — required under GDPR Article 30).
  4. Publish a GDPR-compliant Privacy Policy at /privacy-policy. Must list: data categories, legal basis, all processors, retention periods, data subject rights.
  5. Publish Terms of Service at /terms-of-service.
  6. Activate /gdpr-data-request with a working form.
  7. Configure Zapier Workflow 5 (12-month auto-deletion) before processing any live cases.
  8. Set all Google Drive case folders to "Specific people only". Never "Anyone with the link".
  9. Configure Airtable field and view permissions — notaries see only their own cases.

17.  Final Note

This architecture uses tools that exist, in ways they were designed to be used. No tool is being pushed beyond its limits. No tool is being asked to do another tool's job.

1
Clients never see a third-party tool.

Every client touchpoint is Squarespace. The platform feels like a single product — because for clients, it is.

2
Airtable is an operations tool, not a platform feature.

Notaries use it directly as they would use email. The Squarespace hub is their professional front door.

3
The automation is the product.

The right WhatsApp message at the right moment, sent automatically. That is what clients experience as quality and care. It requires no human. It never misses.

Each component has a defined upgrade path. The architecture can scale without a rebuild.

Annex E — Configuring Zapier Workflows

Before starting: connect Squarespace (forms), Airtable, Google Drive, Gmail, Stripe, and Twilio/WhatsApp to Zapier. Go to Zapier → Apps → + Add connection. Each app requires you to log in and grant permissions.

Workflow 1 — Case Creation (Stripe Trigger)

Trigger

App: Stripe → Event: Checkout Session Completed (Instant) → Connect your Stripe account → Test with a completed test-mode checkout session.

Actions (in order)

  1. Find pending Airtable record: App: Airtable → Find Record → Base: NaaS Platform → Table: Cases → Search by: Client Email → Search value: customer email from Stripe trigger. This locates the pending record created from the form submission.
  2. Generate access code: App: Zapier Utilities → Generate Random String → 8 characters, alphanumeric.
  3. Update Airtable record: App: Airtable → Update Record → Record ID from step 1 → Set: Status = Submitted, Payment Status = Paid, Stripe Payment ID = Checkout Session ID, Advance Amount Paid = amount, Payment Date = today, Client Access Code = from step 2.
  4. Create Google Drive folders: App: Google Drive → Create Folder → Parent: Cases → Name: Case ID. Repeat to create Client-Uploads and Prepared-Documents subfolders.
  5. Send WhatsApp: App: Twilio (WhatsApp) → Send Message → To: client phone number → Body: "Your case [Case ID] is open. Track it here: acte.online/case-[ID]?password=[Access Code]"
  6. Send fallback email: App: Gmail → Send Email → To: client email → Subject: "Your Case is Open — [Case ID]" → Body: same content in branded HTML template.
  7. If notary office pre-selected: App: Gmail or Twilio → notify the assigned office.
Form data bridging: A separate Zapier workflow (triggered by Squarespace form submission) creates a "pending" record in Airtable with Status = "Pending Payment". This stores all form data. When the Stripe webhook fires in Workflow 1, Zapier finds this pending record by client email and upgrades it to a full case record.

Workflow 2 — Status Change → Client Notification

Trigger

App: Airtable → Updated Record → Table: Cases → Watch field: Status.

Actions

  1. Filter by Zapier: only continue if Status has changed.
  2. Paths by Zapier — three branches:
    • Path A (In Review): WhatsApp + email: "Your case is being reviewed and prepared by [Office Name]."
    • Path B (Booked): WhatsApp + email: "Your appointment is confirmed: [Date], [Time], [Address]."
    • Path C (Completed): WhatsApp + email: "Your case is complete. Download your documents: [link]"

Workflow 3 — Booking Confirmation

Trigger

App: Airtable → Updated Record → Table: Cases → Watch field: Booking Date.

Actions

  1. Filter: only continue if Booking Date is not empty.
  2. WhatsApp to client: confirmed appointment date, time, and office address.
  3. Fallback email with the same content.

Workflow 4 — Appointment Preference Captured

Trigger

App: Squarespace → New Form Submission → select the appointment preference form on /thank-you.

Actions

  1. App: Airtable → Find Record → find by client email.
  2. App: Airtable → Update Record → write preferred dates and time preference to the Preferred Date Range field.

Workflow 5 — GDPR Data Retention

Trigger

App: Schedule by Zapier → Every Day → 02:00.

Actions

  1. App: Airtable → Search Records → Status = Completed AND Submission Date older than 365 days.
  2. For each result: Delete Airtable record.
  3. For each result: Delete corresponding Google Drive folder.
  4. For each result: Create a record in the Deletion Audit table — Case ID, deletion date, reason. No personal data.
Technical note: "Looping by Zapier" (for processing multiple records per run) requires Zapier Professional plan. On Starter, only one record is processed per daily run. This is acceptable at MVP. Upgrade to Professional before your case backlog approaches 12 months old.

Workflow 6 — Automated Case Page Creation (Optional, MVP+)

Trigger

App: Airtable → New Record → Table: Cases.

Actions

  1. App: Webhooks by Zapier → POST request to Squarespace Pages API → Duplicate template case page. Full API documentation at developers.squarespace.com.
  2. App: Webhooks by Zapier → PUT request → Set URL slug to Case ID. Set password to Client Access Code.
Technical note: Requires Squarespace API access (Business plan) and comfort with API requests. Not required at launch. Add at 30+ cases/month.
